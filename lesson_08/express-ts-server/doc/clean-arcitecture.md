# Clean architecture
layer - слой
responsibility - у каждого слоя своя ответственность

- domain (entity) - c чем мы раотаем, с какой сущностью(типы)
- repository (persistant layer) - хранение(настройка БД), чтение, запись в БД и изминение БД
- service (business logic) - слой отвечающий за юизнес логику (if else)
- controller - обработка информации

router -> controller -> service -> repository

## Главные преимущества
- можно заменять инструменты и библиотеки без переписывания бизнес логики
- легкое тестирование (testability) - можно тестировать бизнес логику без UI
- изменение в одном слое не сломают код в другом
- разные программисты могут одновременно писать UI и логику
- структура, которая помогает новичкам быстрее понять где лежит нужный код
